---
title: "Seed and config"
description: "How to supply initial data and optional config when creating a provider snapshot."
---

When you call `POST /providers/{slug}/create-snapshot`, you can pass `seed` and `config` in the JSON body.

## Seed

`seed` is a JSON object that becomes the mock provider’s initial state.

### Wrapped form (recommended)

Top-level key must match the provider slug:

```json
{
  "seed": {
    "salesforce": {
      "Account": [
        {
          "Id": "001ACCOUNT000001",
          "Name": "Acme Corp",
          "Industry": "Technology"
        }
      ],
      "Contact": [
        {
          "Id": "003CONTACT000001",
          "FirstName": "Ada",
          "LastName": "Lovelace",
          "Email": "ada@acme.example",
          "AccountId": "001ACCOUNT000001"
        }
      ]
    }
  }
}
```

Object collections under the slug use the provider’s resource names (Salesforce sObject API names, Jira entity names, and so on).

### Bare form

If you omit the slug wrapper, the API wraps it for you:

```json
{
  "seed": {
    "Account": [
      { "Id": "001ACCOUNT000001", "Name": "Acme Corp" }
    ]
  }
}
```

That is equivalent to `{ "salesforce": { "Account": […] } }` when `slug` is `salesforce`.

### Default

If you omit `seed` (or send `null`), the snapshot starts with an empty seed for that provider: `{ "<slug>": {} }`.

## Config

`config` is an optional JSON object on the same request:

```json
{
  "ttl": "6h",
  "seed": { "salesforce": {} },
  "config": {}
}
```

<Note>
  `config` is accepted on the create-snapshot request for forward compatibility with emulator overrides. It is not applied to the running snapshot yet. Pass `{}` or omit the field until config support is enabled.
</Note>

## TTL

`ttl` controls how long the snapshot stays up. Default is `6h`.

| Example | Meaning |
|---------|---------|
| `15m` | 15 minutes |
| `1h` | 1 hour |
| `6h` | 6 hours (default) |
| `1d` | 1 day |

Maximum is `7d`. Invalid values return `400`.

## Full example

```bash
curl -sS -X POST https://simulation.ressl.ai/providers/salesforce/create-snapshot \
  -H "Authorization: Bearer rsk_..." \
  -H "Content-Type: application/json" \
  -d @- <<'EOF'
{
  "ttl": "2h",
  "seed": {
    "salesforce": {
      "Account": [
        {
          "Id": "001ACCOUNT000001",
          "Name": "Acme Corp",
          "Industry": "Technology",
          "AnnualRevenue": 50000000
        }
      ]
    }
  },
  "config": {}
}
EOF
```
